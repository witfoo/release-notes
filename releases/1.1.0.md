# WitFoo Analytics 1.1.0

## Summary

WitFoo Analytics 1.1.0 is a feature release that brings the platform to your phone and tablet, deepens
vulnerability and threat‑model reporting, adds compliance certification packages, and expands
collaboration across your team and the wider WitFoo CyberGrid community — all while hardening the
delivery pipeline behind the scenes. This is a recommended upgrade for all 1.0.x deployments.

**▶ Watch the 1.1.0 tour:** https://vimeo.com/1209183185 — a short walkthrough of the new features.

## What's New

### Mobile & Experience
- **Responsive UI across Analytics, Conductor, and Console.** Every primary workflow now adapts to
  phones and tablets — the navigation collapses to a drawer below tablet width, tables and modals
  reflow, and touch targets are sized for fingers. Investigate and respond from any device.
- **Redesigned Node Details.** A verdict‑first layout leads with the plain‑language answer
  (“Suspicious · 39.1% · Medium”), shows each fact once, and tucks raw JSON and schema behind an
  Advanced disclosure — no more leaked database IDs, floats, or `0001‑01‑01` timestamps.
- **In‑modal relationship pivots.** Node details now include an Edge Search and a “Top relationships”
  view (internal users first), so you can pivot through the security graph without leaving the asset.

### Vulnerability & Threat Management
- **Vulnerability Management report.** Hosts are ranked by severity‑weighted CVE exposure, with
  per‑CVE detail sourced from an offline CVE library (no external calls) and drill‑downs from findings,
  nodes, and reports.
- **Threat Model report.** A self‑contained controls × MITRE ATT&CK matrix built from your audit
  findings, with an embedded AI summary.
- **CVE details, offline.** Click any CVE to see its description and weakness mapping — served from a
  committed, air‑gap‑safe CVE library with a cache‑first lookup.

### Compliance & Certification
- **Certification Packages.** Assemble SOC 2, ISO 27001, and CMMC evidence packages, attach control
  evidence, and export a complete, integrity‑hashed ZIP (report + attachments + manifest) streamed on
  demand.
- **Expanded auditor coverage** and **framework mappings** (PCI DSS v4.0, Essential Eight) across the
  compliance reports.
- **Report Scheduler.** Define a schedule (time range + report set + delivery channels + cadence),
  render reports to offline PDF, and have them delivered automatically by email, Slack, or webhook.

### AI & Automation
- **AI provider enhancements.** Per‑provider model catalogs with best‑practice defaults and an economy
  tier, pre‑save model interrogation (your key is never logged), and policy‑driven routing that sends
  each purpose to the right model — with an economy mode that caps token spend on summaries.
- **Optional MCP authentication.** Secure the Model Context Protocol bridge with a bearer/API‑key at
  the WebSocket handshake, per‑organization and default‑off, with keys hashed at rest.

### Collaboration
- **User profiles** with self‑hosted avatars, a Security/Preferences tab (self‑service password + locale),
  and profile‑linked chat (start a direct message from any author).
- **Chat upgrades:** working file attachments, emoji reactions, and a more resilient realtime connection.
- **CyberGrid community.** Share work units and collections to the CyberGrid directory (SSRF‑safe,
  with a durable retry outbox), browse a community user directory, and opt in to directory federation.

### Detection & Pipeline
- **Richer work‑unit search and collections.** Search by client IP across source/destination, and build
  collections on hostnames, URLs, file names, and file hashes — not just IP.
- **Internal‑user classification** now derives “Internal + Managed” only from real identity sources
  (Entra/AD/Okta/Ping…), so `sshd`/`pam` principals are no longer mislabeled.
- **Graph accuracy.** Host‑authenticated events now correctly mint their internal server/victim node,
  and new NSX‑T parsers classify VMware NSX events instead of dropping them to Unknown.

### Health & Operations
- **Accurate health dashboard.** Total‑memory now reports host memory (not cache‑inflated per‑container
  sums), a Data Retention card shows the *enforced* Cassandra TTLs with oldest‑record age, a per‑node
  Cassandra health monitor surfaces expansion recommendations, and node Online/Offline is derived from
  real last‑seen activity.
- **Conductor dashboard resilience.** Dashboard tiles no longer spin forever — they resolve to data,
  empty, or an error‑with‑retry, with surfaced errors and stale‑while‑revalidate.
- **Scenario‑driven demo data** generates coherent, MITRE‑aligned attack scenarios for realistic demos.

### Platform & Hardening
- **Improved automated monitoring** with detection‑preserving log redaction at the source, opt‑in
  sanitized support packs, and a node → intel problem‑report path.
- **Numerous fixes** across notifications (channel creation), work‑units pagination, faceting, the AWS
  and Cisco Umbrella auditors, SAML login defaults, hardened attachment filename handling, and more.
- **Delivery‑pipeline hardening** turns recurring release‑process risks into automated guardrails
  (pre‑commit hooks, static guards, contract tests) and rescopes the release test suite to run reliably
  in GitHub CI while the exhaustive validation continues on live lab and beta appliances.

## Upgrade Notes

- **No breaking changes.** 1.1.0 is a drop‑in upgrade from 1.0.x.
- New capabilities (MCP auth, directory federation, scheduled reports) are **opt‑in and default‑off**;
  existing behavior is unchanged until you enable them.
- Ephemeral‑container deployments pick up new Cassandra tables automatically via self‑healing schema
  migrations — no manual migration step.
