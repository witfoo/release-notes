# WitFoo Analytics 1.1.0

## Summary

WitFoo Analytics 1.1.0 builds on the 1.0.0 GA (June 2026) with 1.1.0's throughline — *from signal to
evidence and decisive action*: sharper investigations, audit-ready compliance including the new
**Certify** capability, scheduled report delivery, broader data onboarding, and a hardened,
verified multi-tenant foundation. This is a recommended, drop-in upgrade for all 1.0.x deployments.

**▶ Watch the 1.1.0 tour:** https://vimeo.com/1209183185 — a short walkthrough of the new features.

## What's New

### Investigation — sharper, more decisive triage
- **One-click CVE explanations.** Click any CVE in a report or on a host to see a full, plain-language
  explanation — severity, description, and weakness type — served entirely offline from a committed CVE
  library, with no internet calls.
- **Exploitability-aware threat scoring.** A live vulnerability on a host connected to an outside asset
  now raises incident suspicion and flags attacker and victim, and a new "Weak Edge" view (CVE ×
  technique × attacker × victim, with a jump to work units) toggles alongside Attack Chain.
- **Verdict-first entity detail.** The node/entity panel now leads with a clear verdict
  (e.g. "Suspicious · 39% · Medium"), followed by why-flagged / where-who / related / activity —
  values humanized, with raw JSON and schema tucked behind an Advanced disclosure (no leaked database
  IDs, floats, or `0001-01-01` timestamps).
- **In-panel relationship pivoting.** Search all of an entity's relationships from its panel and see a
  "Top relationships" shortlist (internal users first) without leaving the asset.
- **The true target of an attack is always visible.** The internal host under attack (e.g. a server
  being brute-forced) is no longer missing from a work unit's evidence and graph; attacker→victim is
  correctly linked.
- **Readable detail-panel controls.** Fixed detail-panel buttons that could render an unreadable white.

### Asset & Identity — a more trustworthy inventory
- **Accurate internal-user classification.** A user counts as internal/managed only when sourced from a
  real identity system (Entra ID, AD, Okta, Ping…); log noise and failed SSH/PAM logins no longer
  inflate the count, restoring a trustworthy denominator for account-management controls.
- **Open-port visibility on assets.** Assets now carry the network services (open ports) observed on
  them, searchable and filterable from asset search (e.g. "which assets expose remote-desktop access").
- **Full-scope range filtering.** Min/max filters for an asset's port, vulnerability, and product
  counts now narrow the entire dataset, not just the rows already on screen.

### Compliance & Certify — continuous monitoring becomes evidence auditors accept
- **Certify — audit certification packages (SOC 2 / ISO 27001 / CMMC).** Build an audit submission
  inside WitFoo: guided framework questions, attached supporting documents (malware-scanned), answers
  auto-populated from live data (findings, coverage, retention, RBAC, health), exported as one
  integrity-hashed, verifiable package (report + attachments + manifest) streamed on demand. Available
  on Pro/Max tiers.
- **One Auditor, many frameworks.** The Auditor extends beyond CIS Controls; this release adds
  **PCI DSS v4.0** and **Essential Eight** mappings.
- **ISO 27001 updated to the 2022 standard.** Built-in ISO 27001 moves off the withdrawn 2005 edition
  to the current :2022 Annex A (93 controls) — the version auditors actually certify against.
- **Honest compliance-readiness score.** Partial control coverage now counts as partial; a
  not-fully-met control no longer registers as a full pass, so the readiness percentage reflects reality.

### Reporting & Data — reports that arrive on their own
- **Scheduled report delivery.** Schedule reports and dashboards as offline PDFs delivered on a cadence
  to your notification channels — email gets the PDF attached; Slack and webhook get a link and summary.
  No one has to log in to generate them.
- **Retention-ceiling visibility.** The health dashboard now charts how old the oldest record is versus
  the enforced retention limit, per data type, so operators can plan against their ceiling.

### Integrations — broader onboarding, honest views
- **LevelBlue / USM Anywhere ingest.** Pull LevelBlue (AlienVault) USM Anywhere alarms and events
  directly into WitFoo via one API credential per tenant — MSSPs onboard clients with zero endpoint or
  agent changes.
- **VMware NSX-T log coverage.** Correctly recognizes and classifies NSX-T log sources that had been
  flooding the "unknowns" bucket.
- **Accurate Integration Health.** A raw log-server listener is no longer shown as an API-pull
  integration; only genuine integrations appear.
- **Self-monitoring of ingestion quality.** WitFoo's automated monitoring flags data-classification
  anomalies and opens a tracked issue on its own — the mechanism that surfaced the NSX-T gap above.
- **vCenter Broker telemetry parser** — added and queued for human sign-off before it is switched on.

### Reliability & Demo — dependable in production, compelling in the sales cycle
- **Dependable operations dashboard.** The Conductor dashboard no longer hangs on endless spinners — it
  shows data fast, fills charts from the last known-good view, marks stale data, and offers Retry on
  failure.
- **Realistic, story-driven demo.** Replaces random demo noise with a coherent scenario built around a
  believable regional-healthcare organization facing realistic, technique-mapped attacks.

### Trust & Quality — protecting customer data and the platform itself
- **Tamper-proof cross-cluster threat intel.** Shared threat-intel messages between clusters are now
  cryptographically sealed end to end (body, not just labels), rolled out so mixed-version clusters keep
  working during upgrades.
- **Verified access-control enforcement.** Automated safeguards confirm every part of the platform
  enforces the correct permissions, and remaining gaps were closed.
- **New permissions reach upgraded customers.** New permissions (Certify, graph, automation) now
  converge to already-deployed environments on upgrade — no admin hand-editing of roles.
- **Quality & multi-tenant isolation hardening.** A structured program closed test-suite blind spots —
  most notably strengthening one-tenant-cannot-see-another isolation — plus coverage, mobile,
  accessibility, and resilience checks.
- **Delivery-pipeline hardening.** Recurring release-process risks were turned into automated guardrails
  (git-enforced pre-commit hooks, static guards, contract tests) and the release test suite was
  rescoped to run reliably in CI while exhaustive validation continues on live lab and beta appliances.

## Upgrade Notes

- **No breaking changes.** 1.1.0 is a drop-in upgrade from 1.0.x.
- New capabilities (Certify, scheduled reports, the USM Anywhere connector, the vCenter Broker parser)
  are **opt-in** and do not change existing behavior until you enable them.
- Ephemeral-container deployments pick up new Cassandra tables automatically via self-healing schema
  migrations — no manual migration step.
- New permissions (Certify, graph, automation) converge to existing roles automatically on upgrade.
