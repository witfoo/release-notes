# WitFoo Parsers v2026.06.25-203756

**Release Date:** 2026-06-25

## Summary

This release expands WitFoo Parsers from 249 to 270 supported log formats, adding new coverage for Elastic Winlogbeat ECS JSON logs and VMware ESXi/vSAN host logs. Microsoft Defender for Office 365 signal classification now detects phishing click events alongside broadened phishing email detection. Several quality improvements harden clustering logic and message-type contract enforcement following adversarial review cycles.

## What's New

### Features

- **Elastic Winlogbeat ECS JSON Parser** — New parser for ingesting Windows event logs forwarded via Elastic Winlogbeat in ECS (Elastic Common Schema) JSON format, enabling structured processing of Windows security signals through the pipeline. [#266]
- **VMware ESXi/vSAN Host-Log Coverage** — New parsers for VMware ESXi and vSAN host-level logs (`vmwareesxi`, `vmwarehostd`), expanding infrastructure visibility for virtualization environments. [#266]
- **Microsoft Defender O365 Phishing Click Classification** — The `microsoftdefender` parser now classifies `phishing_click` events from Defender for Office 365, providing detection of user interaction with phishing URLs in addition to broadened `phishing_email` signal coverage. [#255]

### Improvements

- **Connector Pipeline Enhancement** — Updates to `connector.go` supporting the new parser integrations and signal classification logic.

### Bug Fixes

- **Clustering Over-Merge Guard** — Fixed an issue where the clustering stage could incorrectly merge distinct log events, improving signal fidelity. Identified and remediated through adversarial review. [#266]
- **Message-Type Contract Test Enforcement** — Strengthened message-type contract validation with click-event anchoring to prevent misclassification regressions. Identified and remediated through adversarial review. [#255]
- **Test Honesty Corrections** — Resolved test assertions that did not accurately reflect expected parser behavior, ensuring test results faithfully represent production correctness. [#266]
