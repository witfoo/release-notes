# WitFoo Console v1.6.0

**Release Date:** 2026-03-03

## Summary

Console 1.6.0 adds pipeline Sankey visualization for conductor data flow monitoring. A new backend API endpoint exposes pipeline flow metrics, and the frontend provides an interactive 5-slice Alluvial/Sankey chart for visualizing signal processing pipelines across conductor nodes.

## New Features

### Pipeline Sankey Visualization (PR183)
- **Pipeline Metrics API**: New `GET /api/v1/nodes/:uuid/pipeline-metrics` endpoint extracts `pipeline_flow_count` metrics from node status history and returns structured flow data
- **PipelineSankeyChart component**: Reusable 5-slice Alluvial/Sankey chart showing data flow from Input Sources through Parsed/Unknown, Stream Names, Kept/Discarded, to Output Connectors
- **PipelineView component**: Conductor node selector with auto-refresh (30s polling), supporting conductor and AIO node roles
- **Pipeline tab on Nodes list**: New tab on the nodes list page provides a dedicated pipeline visualization view with conductor node selection
- **Monitoring tab upgrade**: Node detail page Monitoring tab now uses the new PipelineSankeyChart component, replacing the legacy inline AlluvialChart implementation
- **Color-coded node categories**: Pipeline stages use distinct colors — blue for intake, teal for parsing, cyan for streams, green for kept, red for discarded, purple for outputs

### Backend
- **Pipeline metrics service layer**: `GetPipelineMetrics()` service method with RBAC enforcement (staff+ access), organization scoping, and structured error handling
- **Domain types**: `PipelineFlowMetric` and `PipelineMetricsResponse` domain types for type-safe metric extraction
- **Handler tests**: 5 handler-level tests covering success, invalid UUID, node not found, unauthorized access, and empty metrics scenarios
- **Service tests**: 8 service-level tests including org-scoped access, multiple status history entries, malformed JSON handling, and missing metrics

### Frontend
- **Sankey data transformation**: `pipelineSankey.ts` utility with `buildSankeyFlows()`, `buildSankeyNodes()`, and `buildSankeyColorScale()` functions
- **Carbon AlluvialChart integration**: Dynamic import for SSR safety with DOM color overrides via `$effect` to work around Carbon's hardcoded black node fills
- **E2E tests**: 6 Playwright tests covering tab display, navigation between Nodes/Pipeline tabs, and pipeline chart rendering on node detail pages

## Improvements

### Code Quality
- Deprecated legacy `transformToPipelineSankeyData()` utility in favor of the new `PipelineSankeyChart` component
- Eager pipeline metrics loading on node detail page (previously lazy-loaded only on Pipeline tab click)
- Data-resilient E2E assertions with graceful fallback for tests depending on conductor node availability
