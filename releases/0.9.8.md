# WitFoo Analytics 0.9.8

**Release Date:** 2026-06-23

## Summary

WitFoo Analytics 0.9.8 is a large feature-and-hardening release that builds on the 0.9.7 security baseline. It brings AI autonomy to Work Unit investigation, makes Custom Dashboards and faceted search trustworthy on real data, corrects key Reporter and compliance calculations, and adds a new class of operational-resilience guards so derived state (compliance snapshots, AI spend) can no longer drift after a redeploy. It also resolves a broad set of platform and management-console security findings and completes the consolidation of the codebase into a single monorepo for faster, more reliable releases.

**Feature and security-hardening release — upgrade recommended.**

## What's New

### Security & Hardening

This release resolves a class of log-injection and request-forgery (SSRF) findings across both the analytics platform and the management console, tightens access controls, and refreshes dependencies. No exploit details are published here; the categories below describe the protections now in place:

- **Outbound-connection protection** — Outbound integrations (webhooks, chat notifications, sync connectors) now validate every destination against an anti-rebinding/SSRF guard before any credentials are attached, rejecting private, loopback, and cloud-metadata targets.
- **Tightened access controls** — Per-route permission enforcement was added across the incident and configuration APIs, and a build-time guard now prevents any new route from shipping ungated.
- **Log hygiene** — User-controlled values are sanitized before being written to logs throughout the platform and the management console.
- **Stronger transport validation** — Appliance-to-Conductor health checks now perform full TLS certificate-chain validation instead of trusting any peer.
- **Recurrence prevention** — New automated build-pipeline gates (secret scanning, static analysis, dependency and image scanning) keep these classes of issues from silently re-accumulating.
- **Supply-chain updates** — Vendored dependencies were refreshed to pick up upstream security fixes.
- **Single sign-on reliability** — Azure / SAML SSO assertion-replay handling and request binding were corrected so valid logins are no longer falsely rejected.

### Operational Resilience

0.9.8 hardens the places where derived or runtime state could silently drift from its source of truth after a restart or redeploy:

- **Compliance Readiness no longer strands at 0%** — A snapshot readiness guard preserves the last healthy compliance figure and refuses to overwrite it with a collapsed one, and reporting now waits for graph data to be ready on startup before generating. A redeploy can no longer leave a stale 0% compliance reading in place.
- **AI cost optimization** — Prompt caching, cheaper model tiering for report and on-demand summaries, and demo-mode containment dramatically reduce AI spend while leaving interactive analysis on the operator's chosen model.
- **Production Dashboards** — Custom Dashboard widgets now render real production data or an explicit empty state, never placeholder sample data in a live tenant. Sample data appears only in the dashboard editor preview or under demo mode.

### Features & UX

- **AI Agent autonomy on Work Units** — A Work Unit, or an individual response task, can now be assigned to an AI Agent. The agent sets the Work Unit to Investigating, works the assigned playbook with live, step-by-step investigation progress shown in the Response tab, completes its tasks, and resolves the ticket when the playbook is finished. Assigning to an AI Agent is available whenever an AI provider is configured, and analysts can still assign tasks to any team member.
- **Reliable, consolidated faceted search** — All five search pages (Signals, Nodes, Edges, Work Units, Work Collections) now share one unified filter sidebar with consistent counts that match the result set, accessible tooltips, and capped-count indicators. Unchecking a facet now clears the corresponding page filter.
- **Reporter fixes** — Return-on-investment is now calculated as Total Protection Value divided by Annual Security Spend, with insurance coverage included in protection value, so ROI stays meaningful even in a zero-incident period. The executive Sankey flow diagram now renders fully in view instead of being pushed below the fold.
- **AI Chat fixes** — Work-unit context now reaches the assistant, the chat pop-out works, and direct messages correctly resolve member names and authenticate.
- **Human-friendly names everywhere** — Node detail now shows real Product and Framework names instead of raw ids, the work-unit activity feed shows status labels and user names, and the "Configuration Auditor" is now simply "Auditor."
- **Signal Search from node detail** — A node's detail view can pivot on a stable node id to find all of that node's signals, which is more reliable than searching on an address that can change over time.
- **Auditor UX fixes** — The findings filter list is now reactive and respects multiple filters at once, the findings pager works, and the loading spinner renders in the correct place.
- **Auditor connector fixes** — Microsoft 365, Google Cloud, and Oracle Cloud (OCI) connectors now add and test successfully.
- **Enhanced work-unit activity** — Richer activity notes plus per-task notes and evidence.
- **Licensing improvements** — Improved licensing.witfoo.com reliability, Dev Mode hidden in non-development deployments, and CyberGrid licensing management.
- **User Profile build-out** — An expanded user profile experience.

### Infrastructure

- **Monorepo consolidation** — The platform's previously split repositories are now a single codebase with change-detected image builds, independent per-product versioning, and fully offline builds. The result is faster, more reliable releases with no customer-visible behavior change.

## Upgrade Notes

- **No new required environment variables** are introduced by this release.
- **No manual database migration** is required — schema additions self-heal on startup.
- **No breaking API changes.**
- AI cost-optimization features (prompt caching and model tiering) are automatic and require no configuration. Routing report/summary AI to a cheaper model tier is optional and configurable under AI Settings.
- The "Configuration Auditor" is renamed "Auditor" in the UI; existing audit data and connectors are unchanged.
