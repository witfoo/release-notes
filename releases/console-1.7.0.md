# WitFoo Console v1.7.0

**Release Date:** 2026-06-17

## Summary

Console 1.7.0 introduces Remote Node Management for administrators to control connected nodes directly from the Console, and a Configuration Generator paired with the new `wfa fetch` command to streamline customer node provisioning. Security-focused improvements include hardened error handling on public endpoints and refined data-node protections.

## New Features

### Remote Node Management (PR389, #160)
- **Node Actions Panel**: Administrators with ORG_ADMIN+ privileges can Start, Stop, Restart, Upgrade, and Pull-Images on connected nodes from the node's Settings tab
- **Cluster-Impact Confirmation**: Roles bearing Data or cluster coordination responsibilities (Data, AIO, AIO-Conductor, Processing) require explicit user confirmation before destructive actions; confirmation gates are fail-closed
- **WebSocket Command Transport**: Node actions ride the existing Console↔node WebSocket, ensuring secure, real-time command delivery and result tracking
- **WFA 2.2.0+ Requirement**: Feature requires WitFoo Agent 2.2.0 or later running on the managed node

### Configuration Generator + wfa fetch (PR388, #244)
- **Pre-Built Customer Configuration**: Administrators pre-build and publish a customer node.json with non-VM fields (trial or existing license) and cluster role answers
- **One-Time Tokenized URL**: Configuration is distributed via a single-use, TTL'd (72 hours), TLS-only URL with rate limiting; enables self-service provisioning without credential sharing
- **Secure Token Lifecycle**: Token is hashed at rest; admin password never transits across the network
- **Encrypted Payload**: Configuration payload is encrypted at rest on the server
- **`wfa fetch` Command**: Customer runs `wfa fetch <url>` to apply the pre-built config and collect only VM-specific fields (disk/network) plus admin password locally, then finalizes; auto-validates token TTL and one-time use constraint

## Improvements

### Security Hardening
- Hardened error handling on public `/fetch` and `/config` endpoints to prevent internal-detail leakage
- AIO-Conductor data-node guard fix to enforce role-based access controls on sensitive node state

### Node Actions Monitoring
- Live job-status polling on the Node Actions panel for real-time feedback on command execution
- Enhanced result tracking and user-facing status updates
