# WitFoo Parsers v2026.06.20-183145

**Release Date:** 2026-06-20

## Summary

This release delivers a comprehensive adversarial-audit remediation across the full parser collection, resolving 4 Priority 1 and 6 Priority 2 issues while hardening defenses against 2 additional Priority 2 attack vectors. The release also completes a significant architectural change by de-submodulizing all parser modules into the main repository, and addresses post-integration regressions in linting, security checks, and submodule test infrastructure. All 267 parsers were touched as part of this effort.

## What's New

### Improvements

- **Adversarial-audit remediation (PR403/404):** Resolved 10 issues identified during adversarial security audit of the parser pipeline:
  - **4 P1 (Critical) fixes:** Addressed high-priority vulnerabilities in parser fingerprinting and signal processing logic
  - **6 P2 (High) fixes:** Corrected secondary issues across regex matching and pipeline handling
  - **2 P2 defenses hardened:** Strengthened existing controls against two additional attack vectors without requiring code changes
- **Repository architecture: de-submodule migration (PR403a):** All parser submodules have been snapshot-imported directly into the main repository. This eliminates submodule synchronization issues, simplifies CI/CD pipelines, and reduces build complexity for downstream consumers.
- **Parser collection expanded to 267 parsers**, covering the full breadth of security log sources from network infrastructure, cloud platforms, endpoint protection, identity management, and application layers.

### Bug Fixes

- **Post-fold inspection regressions (PR404):** Fixed regressions introduced during the de-submodule migration that affected:
  - Linter rule compliance across parser definitions
  - Security validation checks in the parser pipeline
  - Submodule-based test harness compatibility
