# WitFoo Analytics v0.9.2

**Release Date:** 2026-03-03

## Summary

This release introduces pipeline Sankey visualization in the console UI, new data exporters for USM Anywhere webhooks and cloud object storage, and summation pipeline controls in the conductor UI. Significant work was invested in fixing AI-powered report and summary generation — resolving data context, timeout, and type mismatch issues that previously produced empty or inaccurate results. Infrastructure stability improvements address deployment issues with SSR, reverse proxy behavior, and conductor dashboard performance.

## What's New

### Features

**Pipeline Sankey Visualization**
- Added interactive Sankey diagram to the console UI for visualizing data flow through processing pipelines, including stream connectivity, flow normalization, top-10 stream aggregation, and classification labeling

**USM Anywhere Webhook & Cloud Object Storage Exporters (PR181 + PR182)**
- New exporter for forwarding data via AT&T USM Anywhere webhook integration
- New exporter for writing to cloud object storage targets

**Conductor Pipeline Controls (PR179 + PR180)**
- Added summation domain type and artifact-filter wiring with a toggle in the conductor UI for enabling/disabling summation messages
- Added discard export capability with dedicated UI and Sankey visualization support, including source_subject exporter support

### Improvements

- Enriched AI report system prompts with WitFoo domain terminology for more accurate, context-aware output
- AI work unit summaries now include node details, accurate counts, and actual data context for meaningful analysis
- Increased reverse proxy timeout to 3 minutes for AI endpoints to accommodate longer inference times
- Included MITRE ATT&CK taxonomy data in the development seeder for richer local testing
- Multiple dependency updates across all services, including common library bump to v1.5.11

### Bug Fixes

**AI Summary & Reporting**
- Fixed `severity_level` type mismatch (string vs. int) that caused empty AI summaries
- Corrected work unit AI summary to use the detail endpoint instead of the list endpoint, which was returning the wrong incident
- Resolved AI summary timeouts for report generation

**Deployment & Infrastructure**
- Disabled server-side rendering (SSR) for all pages, fixing 502 errors on load-balanced deployments
- Fixed reverse proxy redirect handling — upstream redirects are no longer followed, and trailing slashes are preserved
- Corrected API default incident-engine hostname to use `.lan` TLD
- Updated environment variables for reverse proxy and Docker Compose configurations

**Conductor & UI Stability**
- Fixed conductor dashboard browser lockup caused by timer leaks, unbounded chart re-renders, and unthrottled ResizeObserver callbacks — applied memoization and cleanup
- Fixed Sankey flow normalization, exit node rendering, and stream connectivity issues
- Resolved auto-parser `DetectFormat` regression in the conductor suite
- Fixed Vite font-face stripping issue and reporter cleanup in the framework seeder

**Data & Testing**
- Fixed `org_id` mismatch and invalid `status_id` in demo data pipeline seeder
- Resolved flaky `TestGenerateOrgID_Uniqueness` test in the licensing API
- Fixed multiple E2E test reliability issues: login form hydration waits, button selector matching, test timeouts under load, status history handling when no data is persisted, and TypeScript type errors in spread arguments

## Breaking Changes

- **SSR disabled globally:** Server-side rendering is now disabled for all UI pages. Deployments relying on SSR behavior (e.g., SEO or pre-rendering) should verify compatibility.
- **Reverse proxy redirect behavior changed:** The reverse proxy no longer follows upstream redirects. Environments with custom redirect chains through the proxy should be tested.
- **API default hostname changed to `.lan` TLD:** The incident-engine default hostname now uses `.lan`. Deployments using the previous default without explicit configuration should update their DNS or environment variables accordingly.
