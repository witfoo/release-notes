# WitFoo Analytics v0.9.2

**Release Date:** 2026-03-02

## Summary

WitFoo Analytics v0.9.2 delivers 30+ feature PRs spanning cloud object storage exporters, AI-powered playbooks, pipeline visualization, reporter drill-down navigation, TOFU trust management, conductor SSO, and significant UI improvements. This release includes 328 commits since v0.9.1 with comprehensive enhancements to the signal processing pipeline, data export capabilities, and user experience across analytics, conductor, and console components.

## What's New

### Features

**Cloud Object Storage Exporters (PR182)**
- Three new artifact export destinations: AWS S3, Azure Blob Storage, and Google Cloud Storage
- NDJSON format with gzip compression and time-partitioned paths (`{prefix}/{org_id}/YYYY/MM/DD/HH/{timestamp}-{batch_id}.ndjson.gz`)
- S3 supports custom endpoints and path-style addressing for MinIO, R2, B2, and Wasabi compatibility
- Azure supports connection string and shared key authentication
- GCS supports service account JSON and Application Default Credentials
- Default batch of 100 artifacts with 60-second flush interval

**USM Anywhere Webhook Exporter (PR181)**
- New `usm-anywhere` connector for AT&T Cybersecurity USM Anywhere integration
- JSON array batching (default 50, max 10,000) with gzip compression
- API key authentication via `API_KEY` header
- Full Sankey flow metrics and source subject resolution support

**AI-Powered Playbooks (PR175)**
- Agentic AI analysis steps in the playbook system for automated incident investigation
- `AIExecutor` with Claude integration using `SecurityToolExecutor` (search_artifacts, search_nodes)
- Auto-triggered via criteria: suspicion score >= 0.50 and status not resolved
- Rate limited per organization (30-second cooldown)
- Auto-configured Claude Opus 4.6 provider from `.netrc` credentials
- Seeded AI playbooks with frontend types and i18n support

**ProtoGraph Summation Messages (PR180)**
- Summation domain type with artifact-filter wiring in conductor pipeline
- Summation toggle in conductor-ui for configuring aggregation behavior

**ProtoGraph Discard Export (PR179)**
- Discard export pipeline with `DedupeWithDiscard` fan-out pattern
- Source subject resolution across all 8 exporters without modifying Settings structs
- Sankey visualization for discard flows in conductor-ui

**Pipeline Sankey Visualization (PR177)**
- 5-column Sankey diagram showing real-time pipeline data flow
- Top-10 stream aggregation with proper label and classification
- Flow normalization, exit node handling, and stream connectivity fixes

**Reporter Drill-Down Navigation (PR171)**
- Click-through navigation from reporter metrics to detailed views across all 4 report pages
- Bar chart and donut slice click events wired via Carbon Charts `bind:chart`

**System Capacity & License Enforcement (PR168)**
- `MaskLicenseKey()` with PCI DSS masking pattern
- License checker graceful degradation (stores "unavailable" on error)
- 6 new Cassandra columns for license diagnostics
- Capacity history LineChart, InlineNotification diagnostics, 16 i18n keys

**TOFU Trust Reset & Remote Job (PR173)**
- Three trust stores: TLS `trust.json`, peer certificates, SSH `known_hosts`
- `TrustStore.ResetAll()` with backup creation and timestamps
- CLI commands: `wfa trust {list, show, reset, reset-all}` with `--force` flag
- Remote job type `trust_reset` via licensing API

**Conductor SSO Auth & Menu Visibility (PR165)**
- `SecondaryNetworks` on ContainerConf connecting reverse-proxy to conductor-net
- `/conductor/status` unauthenticated endpoint for NavBar visibility gating
- Shared JWT secret between analytics proxy and conductor-ui
- Docker dual-network compose configuration

**Pipeline Performance & Index Optimization (PR166)**
- SAI (Storage Attached Index) creation for efficient query lookup
- Per-org iteration pattern replacing cross-org full-table-scan queries
- HDR Histogram for query profiling with microsecond precision
- Conductor GOMEMLIMIT set to 90% of container memory limit

### UI Improvements (PR178)

- **Navigation Restructure (PR178c)**: Reorganized navigation for improved workflow
- **Chat Overhaul (PR178d)**: Slack-inspired chat with emoji reactions, user presence indicators, default Lobby room, emoji picker, and @mention improvements
- **Profile System (PR178e)**: Chat maximize/detach, account profile persistence via Cassandra `user_profiles` table
- **AI Summaries (PR178f)**: AI Summary buttons on all 4 report pages and work collections with Claude Opus 4.6 auto-configuration
- **Search Facets (PR178g)**: Server-side facet counting endpoint, enabled playbooks tab on work unit detail, enable-all endpoints for playbooks and responder jobs
- **Tenant Management (PR178b)**: Tenant purge-duplicates endpoint with friendly 410/422 error handling

### Reporter Quality (PR167)

- Backend data fixes for FTE, control names, and product resolution
- Data seeding fixes for MTTR, daily summary, and verification
- Tooltip enhancements across all 4 reporter pages
- Control-level graph search filtering
- i18n tooltips, seed variance, and verification checks

### Detection Alignment (PR170) & Bug Fixes (PR169)

- Data alignment and inspection cleanup across detection pipeline
- Chat bug fixes: ChatRoomServiceInterface extraction, markRead empty body defense, uploadFile comprehensive fix

## Breaking Changes

None. This release is backward-compatible with v0.9.1 deployments.

## Upgrade Notes

- New cloud object storage exporters (S3, Azure Blob, GCS) require appropriate cloud credentials to be configured in the conductor-ui exporter settings
- AI playbooks require a configured AI provider (Claude Opus 4.6 recommended) — auto-configured from `.netrc` when available
- TOFU trust reset CLI commands (`wfa trust`) require WFA 2.0.30+
- Conductor SSO requires `WF_JWT_SECRET` and `REVERSE_PROXY_MODE=true` environment variables on conductor-ui
