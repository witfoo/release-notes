# WitFoo Parsers v2026.07.15-105320

**Release Date:** 2026-07-14

## Summary

This release expands the parser library from 249+ to 285 parsers, driven by significant new coverage for VMware NSX-T, on-premises identity providers (Okta, PingIdentity), and USM Anywhere log classification. Alongside the new parsers, the CI replay-test infrastructure received extensive hardening to handle the increased parser load reliably, and multiple P1/P2 remediation fixes improve fingerprint accuracy and asset-inventory mapping.

## What's New

### Features

- **VMware NSX-T application-name parsers** — Added dedicated parsers for `nsx`, `nsx-exporter`, `nsx-sha`, `nsx-sha-python`, and `cfgAgent` log sources, enabling granular signal extraction from NSX-T distributed firewall, proxy, operations-agent, and security-health components (#341)
- **VMware NSX Manager TLS/trust parser** — New parser for NSX Manager certificate and trust-negotiation events (#375, #418)
- **VMware ESXi / vCenter coverage widening** — Audit batch (2026-07-08) adds fingerprints derived from fleet-unknown analysis, reducing unclassified log volume from VMware infrastructure
- **On-premises identity parsers** — New parsers for Okta, PingIdentity, and domain-controller-gated Active Directory logs with identity-backfill support, enabling unified identity signal processing across hybrid environments (#324)
- **USM Anywhere parser classifier** — Alarms are now classified as `siem_event` by intent while raw events route to `asset_inventory`, improving downstream signal fidelity for AT&T/LevelBlue USM Anywhere integrations (#344)

### Improvements

- **Registry guards for new parsers** — All new NSX parser keys are registered in the parser registry with bijection and count guards, ensuring build-time detection of key collisions or omissions (#345)
- **Static guards and coverage ratchet** — Property tests, hook overhaul, and untested-connector floor guards strengthen the parser contribution workflow (#345)
- **Replay test naming** — Swallowed replay samples now report their names on processing-wait failure, accelerating root-cause analysis during CI runs (#432)
- **Diagnostic logging on replay mismatch** — Unexpected-unknown message contents are dumped when parser-replay results diverge from expectations

### Bug Fixes

- **Inventory-asset source-identity mapping (P1)** — Corrected asset-to-source identity mapping in the USM Anywhere pipeline and exposed the live lead-rule seam for operator validation (#344)
- **UPN gate and operator-edit stamp (P1)** — Fixed User Principal Name gating logic and operator-edit attribution for identity parsers; added PingIdentity guard and AD attribution corrections (#324)
- **NSX INFO-marker and raw-only fingerprint coverage** — Added missing INFO log-level markers and raw-only fingerprint paths for NSX sibling parsers; includes reconstruction disclosure for audit transparency (#341)
- **Replay validation logic reverted** — Reverted keepalive-flow + TimeID correlation approach for replay validation after regression; burst-count validation restored
- **CI replay pipeline timeouts** — Widened replay timeouts (360s processing window, 600s root) and switched message counting to O(1) stream subject-info to eliminate contention under 280-parser load (#345)
- **CI run-state probe** — Parser-replay run-state probe now reads all available logs instead of truncating at `--tail=1000`, preventing false-positive readiness signals
- **Build toolchain** — Conductor build-broker-edge and artifact-filter builds now force `GOWORK=auto`, resolving module resolution failures in multi-workspace CI environments
- **Code formatting** — Applied `gofmt` to connector and NSX parser source files
