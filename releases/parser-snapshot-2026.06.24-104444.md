# WitFoo Parsers v2026.06.24-104444

**Release Date:** 2026-06-24

## Summary

This release expands the parser library to 269 parsers with new Microsoft Graph Security coverage, introduces a severity-range validation gate to enforce RFC 5424 compliance, and resolves a critical key-value parsing defect that could silently drop log fields containing dot-boundary patterns. Infrastructure hardening includes a Beats agent liveness tracker and improved test resilience under concurrent load.

## What's New

### Features

- **Microsoft Graph Security parsers** — Added two new parsers (`microsoftdirectoryaudit`, `microsoftidentityprotection`) covering previously unaudited Microsoft Graph API event types; fixed field mapping in the existing `riskyUsers` parser within `microsoftentrasignin`.
- **Severity-range validation gate (Gate 8)** — New `quality_checker` gate ensures no parser emits a `SeverityCode` outside the RFC 5424 range `[0, 7]`, preventing invalid severity values from reaching downstream analytics.
- **Beats agent liveness tracker** — New signal-parser pipeline stage that extracts KV pairs and metrics from Beats agent heartbeat events, enabling detection of silent collector failures.

### Improvements

- **RFC 5424 severity guard** — All parsers now enforce strict bounds on `SeverityCode`, rejecting or clamping values that fall outside the `[0, 7]` range. Affected parsers: `f5asm`, `greynoise`, `microsoftdefenderincident`, `nginx`, `proofpoint`, `ufw`.
- **Non-tautological severity gate** — Replaced a previously tautological severity check with a meaningful validation predicate, ensuring the gate actually rejects malformed data.
- **Smoke test hardening** — Signal-parser smoke tests now build against the main workspace binary and include run-state probes resilient to heavy concurrent load, reducing false failures in CI.
- **Dependency remediation** — Vendored/offline Dependabot updates applied for security patches across transitive dependencies.
- **Monorepo cleanup** — De-submoduled documentation and removed stale scripts, simplifying the build and contribution workflow.
- **Code formatting** — Applied `gofmt` across `conductor_suite`, `console-ui`, and `incident-engine` packages for consistency.

### Bug Fixes

- **P0 — Dot-boundary KV-key drop** — Fixed a critical defect in the KV extraction pipeline (`connector.go`) where log field keys containing dot boundaries (e.g., `process.name`, `event.action`) were silently dropped, causing data loss in parsed output. Discovered during pre-deploy verification.
- **P1 — Winlogbeat cardinality regression** — Corrected a cardinality miscalculation affecting Winlogbeat event deduplication.
- **Microsoft Entra sign-in parser** — Fixed field mapping errors in `microsoftentrasignin` that could misattribute risk-level indicators from risky-user events.
