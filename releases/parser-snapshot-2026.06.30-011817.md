# WitFoo Parsers v2026.06.30-011817

**Release Date:** 2026-06-30

## Summary

This release adds production-ready support for VMware NSX-T log parsing by auditing and enabling two new parsers — `nsxopsagent` and `vmwarensxproxy` — bringing the total parser count to 272. Additionally, NSX TLS certificate failure events are now properly escalated as security signals, and infrastructure fixes improve the reliability of the build and test pipeline.

## What's New

### Features

- **NSX-T parser support enabled:** Audited and activated the `nsxopsagent` and `vmwarensxproxy` parsers for VMware NSX-T environments, providing structured log ingestion for NSX operations agent and proxy components. (#293, #295)

### Improvements

- **NSX TLS certificate failure escalation:** TLS certificate failures from NSX components are now escalated as actionable security signals, ensuring certificate-related issues surface appropriately in SOC workflows. (#293, #295)
- **Untested-marker gap closed:** Resolved coverage gaps where certain NSX parser paths were not flagged for testing, improving parser quality assurance visibility.
- **Connector updates:** Changes to `connector.go` and `microsoftdefender` parser configurations to support pipeline improvements.

### Bug Fixes

- **Build pipeline fix:** Corrected the conductor test configuration to properly build `broker-edge` for signal-parser smoke and integration test gates, preventing false build failures.
- **SOC client 204 response handling:** Fixed an issue where HTTP 204 responses from the client caused open issue regression tests to fail incorrectly on SOC environments. (#317)
