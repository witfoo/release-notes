# WitFoo Analytics v0.9.3

**Release Date:** 2026-03-12

## Summary

WitFoo Analytics v0.9.3 delivers SAML onboarding wizard support across all three UIs, pre-release bug fixes, production launch readiness improvements, and comprehensive dependency updates. This release includes 84 commits since v0.9.2 with major enhancements to authentication configuration, deployment stability, and i18n coverage.

## What's New

### Features

**SAML Onboarding Wizard (PR191)**
- Guided 7-step SAML SSO setup wizard with provider-specific presets for Azure AD (Entra ID), Okta, OneLogin, PingIdentity, and Manual configuration
- Provider selection tiles with pre-populated attribute mappings, metadata URL templates, and detailed per-provider setup instructions (5-8 steps each)
- SAML configuration test endpoint validates certificate validity, key pair match, IdP metadata URL accessibility, and SP build
- SP key pair auto-generation: self-signed RSA 2048-bit certificates with 10-year validity
- Wizard available in both onboarding step 8 (Authentication) and admin settings page via "Configure with Wizard" modal
- `mixed_saml` auth mode exposed in UI — enables SAML SSO with local credential fallback
- 89 i18n keys across 7 locales (English, Spanish, French, German, Japanese, Arabic, Te Reo Māori)
- E2E tests (149-150) and system test step 8.24 for SAML validation

**SAML Wizard for Conductor-UI and Console-UI (PR192)**
- Same guided wizard ported to both conductor-ui and console-ui
- Backend SAML test and SP key generation endpoints added to both services
- Conductor-UI: Carbon 0.89.x with `on:click` event syntax, integrated into SAMLSettingsFields
- Console-UI: Carbon 0.101.x with `onclick` event syntax, integrated into settings page
- "Configure with Wizard" button on SAML settings in both UIs

**Te Reo Māori Locale (PR187)**
- 7th UI locale added: Te Reo Māori (mi) with full translation coverage

**Microsoft Sentinel Connector (PR185)**
- New artifact export connector for Microsoft Sentinel integration
- Alphabetical reordering of connector list in conductor-ui

**SAML Auth Fallback (PR186)**
- Cookie-based redirect loop detection prevents infinite SAML config error loops
- `?auth_fallback=true` URL parameter forces local auth display in SAML-only mode
- Implemented across all three UIs (analytics, conductor-ui, console-ui)

**i18n Completion + AI Language Awareness (PR188)**
- Signal search facet i18n with `$derived.by()` pattern for reactive Paraglide arrays
- CyberGrid publications/subscriptions i18n (~72 new keys × 7 locales)
- AI language awareness: `LocaleInstruction(locale)` appends language instruction to AI prompts
- Frontend passes `getLocale()` to summary/chat endpoints for locale-aware AI responses

### Bug Fixes

**Pre-Release Bug Fixes (PR189 — 25 issues across 5 sub-PRs)**
- AI summary cache locale key fix — prevents stale cached summaries across language switches
- Signal search server-side location filter using `pkg/geo.IsInternalIP`
- System health 500 error guard for empty org_id
- Playbook step name JSON field mismatch (`title` → `name`, `action` → `action_type`)
- ATT&CK sub-techniques now show all techniques (not just observed)
- Work unit layout consolidated from 10 to 5 tabs with Accordion
- CyberGrid anonymous date zero-value check
- Button overflow fix with `size="small"`
- Queue chart Carbon g90 theme alignment
- Classification rules CRUD page
- Features settings page with dev tenant seeding
- CyberGrid shared features (work_collections + chat_rooms)
- Conductor feed approval workflow (schema + domain + UI)
- Conductor SSO `WF_JWT_SECRET` fix in dev docker-compose

**Production Launch Readiness (PR190)**
- Cluster startup validation and fixes
- Load balancer and SSL termination hardening
- WFA Docker system prune tests
- Security headers improvements in reverse-proxy

**Cassandra Seeder Consolidation (PR184)**
- `init-schema.cql` converted to DDL-only — all INSERT/UPDATE extracted to seeder source files
- Production seeds auto-loaded on container startup
- 24 historical migration files archived

### Infrastructure

- Dependency updates across all Go modules and npm projects
- WFA bumped to v2.0.36
- All conductor_suite submodules updated and pushed

## Upgrade Notes

- No breaking API changes
- No database migration required (seeder consolidation is backward-compatible)
- `mixed_saml` auth mode is now available in the onboarding wizard and admin settings
- SAML test and key generation endpoints are new additions — no impact on existing configurations
