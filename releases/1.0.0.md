# WitFoo Analytics 1.0.0

**Release Date:** 2026-06-30

## Summary

WitFoo Analytics 1.0.0 is the platform's general-availability (GA) milestone. Building directly on the 0.9.8 security baseline, it completes a broad cycle of feature work, reporting depth, and operational hardening, and graduates the platform to a stable 1.0 release line with independent per-product versioning. Highlights include two new executive-grade reports (Vulnerability Management and Threat Model), a major step up in AI provider control and cost governance, a real collaboration layer (chat upgrade, user profiles, and CyberGrid community sharing), responsive mobile/tablet support across every console, and a dedicated pre-1.0 full code review that raised the security and test-depth bar across the entire codebase.

**General-availability release — upgrade recommended.**

## What's New

### Reporting & Compliance

- **Vulnerability Management report** — A new report ranks hosts by severity-weighted vulnerability exposure, with drill-down from a host to its CVEs and from a CVE to its affected hosts, all server-paginated for large estates.
- **Threat Model report** — A new self-contained report renders your security controls against the MITRE ATT&CK matrix from real audit findings, with per-cell drill-down to the contributing findings and an optional AI narrative summary.
- **Reporter improvements** — Dark-theme charts now render correctly, the Threat Model and Vulnerability views gained interactive drill-down and paging, and the Reporter navigation was reorganized into a clearer order with "Pro" and "Configuration" groupings.

### AI & Automation

- **AI Provider Enhancement** — Each provider now ships a curated model catalog with best-practice defaults and an economy tier; the platform can interrogate a provider's live model list before you save (your key is validated, never logged); and a new AI profile (`best` / `economy`) automatically routes each AI purpose to an appropriate model, with operator pins preserved. Economy mode caps token spend on report and summary generation while leaving interactive analysis on your chosen model.
- **Improved AI tooling** — The AI assistant can now use a set of read-only platform tools to answer questions grounded in your live data.
- **MCP Authentication** — The Model Context Protocol integration gained optional, per-organization authentication (bearer token or API key, default off; keys are hashed at rest and shown once). The existing behavior is unchanged unless you opt in.
- **AI summary caching** — On-demand AI summaries (including Auditor findings) are cached per organization to cut cost and latency.

### Collaboration & Profile

- **User Profile** — A full profile experience: self-hosted avatar upload (no external CDN), self-service password and locale, an author card and presence, and one-click "message a user" from anywhere their name appears.
- **Chat Upgrade** — File attachments, emoji reactions, and a more robust realtime connection (bounded reconnect, presence staleness handling).
- **CyberGrid community** — Share Work Units and Work Collections to the CyberGrid community directory, a within-cluster and cross-cluster User Directory, and a durable, self-healing directory-submission pipeline that reports its true status instead of silently failing.

### Detection & Pipeline

- **Azure Email Security** — Microsoft Defender for Office 365 phishing signals (phishing email and phishing-click) now produce Work Units.
- **New parsers** — VMware NSX-T (`nsxopsagent`, `vmwarensxproxy`) operational and degraded-service events are now classified and actionable.
- **Conductor visibility** — Per-agent up/down and health tracking for Beats agents, expanded integration-health detail in Conductor, and the Conductor link/health is now correctly hidden on non-Conductor nodes.
- **Search & collections** — Work Unit search now matches on client IP across source and destination addresses; Work Collections can be built on hostnames, URLs, file names, and file hashes (not just IPs); and the Work Definition facet, Client-IP search, and successive-page loading were corrected.

### Platform & Operations

- **Mobile & tablet support** — The Analytics, Conductor, and Console UIs are now responsive, with a drawer-based primary navigation and touch-friendly controls below desktop widths.
- **Sign-in** — The login page defaults to SAML single sign-on when it is enabled (local login always remains reachable), and the Azure/SAML SSO reliability issues were resolved.
- **Health dashboard** — A new Data Retention card shows the actually-enforced storage retention, the Total Memory gauge now reports true host memory, and a new per-node Cassandra health view surfaces cluster status and expansion guidance.
- **Improved automated monitoring** — Log/telemetry egress is now redacted at the source with a configurable policy, and appliances can report cluster problems back to WitFoo (opt-in, PII-free) so issues are caught proactively.
- **Console node management** — Analytics nodes can now be managed from the Console, with an accurate online/offline indicator derived from last-seen time.
- **Fixes** — Notification-channel creation, the AWS audit connector, the password-reset flow, and CyberGrid publication deletion were all corrected.

### Infrastructure & Hardening

- **Pre-1.0 full code review** — A dedicated review pass before GA covered external-dependency and vulnerability verification, Cassandra optimization, an adversarial re-review of the entire 0.9.8→1.0.0 change set, deeper test and docker-log coverage, documentation accuracy, and an expanded certification curriculum.
- **Cassandra** — Upgraded to 5.0.8 with a per-node health monitor.
- **Supply chain & CI** — Continued hardening of the offline-build supply chain and the automated security gates (secret scanning, static analysis, dependency and image scanning, log-injection and SSRF call-site gates).

## Upgrade Notes

- **No new required environment variables.** New capabilities — MCP authentication, AI profiles, and telemetry redaction — are **opt-in** and default to today's behavior.
- **No manual database migration** is required — schema additions self-heal on startup.
- **No breaking API changes.**
- AI economy/best profiles and prompt caching are optional cost controls configured under AI Settings; interactive analysis stays on your chosen model.
- Telemetry redaction is configured by an operator-set policy (`off` / `standard` / `strict`); `standard` is pseudonymization, not anonymization.
